Case Study 05 / Content Ops · APIs · Media

Multi-Platform Content Publishing

A publishing engine that turns a content calendar into live posts across Instagram and LinkedIn — images, videos, carousels, and documents — with temporary hosting and automatic cleanup.

GitHub More case studies

Design, build & operate
Stack
n8n · Instagram Graph API · LinkedIn REST API · Supabase · Drive

Impact
Zero manual posting · 2 platforms · 4 formats

01The problem

Publishing across Instagram and LinkedIn was manual, slow, and error-prone, and different content types (image, video, carousel, document) each need a different API dance. The team needed calendar-driven publishing that just worked and left no mess behind.

04Integrations & AI components

  • APIs: Instagram Graph API (POST /media, POST /media_publish), LinkedIn REST API (documents initializeUpload + posts, OAuth2 scopes w_member_social), Google Sheets, Google Drive, Supabase Storage (temporary public hosting).
  • AI: none — this is pure orchestration and media-lifecycle engineering.

05Technical challenges

  • Media-not-ready handling — wait + status check + retry before publishing.
  • Format branching for image / video / carousel / document, each with its own API sequence.
  • Guaranteeing zero orphaned files by deleting temp assets only after a successful publish.
  • LinkedIn OAuth2 scopes and document upload initialization.

06Reliability engineering

Deployment
Self-hosted n8n · schedule every 15 min / hourly
Monitoring
Status = Published + timestamp written to the sheet
Error handling & retries
Upload failure stops + logs + preserves assets · publish failure never deletes assets or marks published · retry on media-not-ready
Validation & scale
Per-row calendar processing · zero orphaned files / zero storage accumulation

08Lessons learned

Publishing is the easy part — the real work is the media lifecycle: temporary hosting, guaranteed cleanup, and retry-on-not-ready so a slow API never leaves a broken post or a storage leak.
