AI Contact Segmentation at 25k Scale

Turned 25,000+ email-only HubSpot contacts into a segmented, targetable database using domain grouping and website-content LLM classification — with a cost gate baked in.

01The problem

HubSpot held 25,000+ contacts with nothing but an email address — no industry or segment — so targeted campaigns were impossible and manual sorting didn't scale.

03Workflow

HubSpot contacts→Group by domain→Fetch website→Clean + truncate→LLM classify 6 cats→Split to contacts→HubSpot segment + lists

04Integrations & AI components

  • APIs: HubSpot (read + custom property + Active Lists), HTTP website fetch, Azure OpenAI GPT-4o.
  • AI: LLM classification into 6 US-market categories with confidence score + reasoning; fallback category logic for inconsistent output.

05Technical challenges

  • Cost control at scale — grouping by email domain avoided redundant company lookups.
  • A webhook variant gated AI to business domains only, skipping free/personal domains.
  • Token-budgeted website truncation.
  • Fallback logic so every contact resolves to a category.

08Lessons learned

Cost is a design decision, not an afterthought. Domain grouping + a business-domain gate cut LLM calls dramatically while keeping quality — the difference between a viable pipeline and an expensive one.
